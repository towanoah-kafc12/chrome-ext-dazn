## 技術選定

### 採用する技術

| 領域 | 採用技術 | バージョン / 条件 | 採用理由 |
| --- | --- | --- | --- |
| 言語 | JavaScript / CSS / JSON | ブラウザ標準 | ビルドなしでChrome拡張として動かせるため |
| フレームワーク | なし | 素のManifest V3 | MVPの目的がDOM/CSS調整であり、フレームワーク不要のため |
| UI | なし | content scriptのみ | ユーザ操作なしで自動適用するMVPにするため |
| 状態管理 | なし | DOM状態から都度判定 | 設定や永続状態が不要なため |
| データ保存 | なし | `chrome.storage` 未使用 | 個人情報や設定値を保存しないため |
| 認証 | なし | DAZN認証には介入しない | 拡張はページ表示だけを調整するため |
| テスト | 手動確認 + 構文確認 | Chrome拡張読み込み、JSON parse、JS構文確認 | DAZNログイン後の実機確認が必要なため |
| デプロイ / 実行環境 | Chrome / Chromium系ブラウザ | Manifest V3対応 | ローカル拡張として読み込むため |

### 採用しない技術

| 技術 | 採用しない理由 | 再検討条件 |
| --- | --- | --- |
| Vite / TypeScript | 初期MVPにはセットアップが重く、確認したいことがCSS/DOM調整中心のため | 設定画面、複数機能、テスト強化が必要になった場合 |
| ReactなどのUIフレームワーク | ポップアップや設定画面を作らないため | 拡張の操作UIが必要になった場合 |
| Chrome Web Store公開対応 | まず個人利用のローカル読み込みを優先するため | 配布・公開する方針になった場合 |

## 実装方針

### アーキテクチャ方針

- `app/` をChromeに読み込ませる拡張ルートにする。
- `app/manifest.json` にManifest V3設定とDAZN向けcontent scriptを定義する。
- `app/content.js` で `video` 要素を検出し、動画プレイヤー周辺の要素へCSSクラスを付与する。
- `app/content.css` で付与されたクラスに対し、横幅最大化と高さ上限のCSSを適用する。
- `app/content.js` でDAZNヘッダーの高さと動画アスペクト比を計算し、ヘッダーを除いた表示可能領域を超えない最大幅をCSS変数として渡す。
- DOM変更と画面リサイズ、動画メタデータ読み込みに追従するため、`MutationObserver`、`resize` イベント、`loadedmetadata` イベントで再適用する。

### データ設計方針

- 永続データは持たない。
- DOMに付与するCSSクラスだけを状態として扱う。
- DAZNのページ内容、認証情報、視聴情報は読み取らず保存しない。

### UI / UX 方針

- ユーザ操作なしで自動適用する。
- 初期MVPではON/OFF UIを作らない。
- ページ全体ではなく、動画プレイヤー周辺の横幅制約を優先的に緩める。
- 画面が崩れすぎる場合は、実際のDAZN DOMを確認して対象セレクタを絞る。

### エラーハンドリング方針

- `video` 要素が見つからない場合は何もしない。
- DAZN側のDOM変更により対象が見つからない場合でも、ページの通常表示を壊さないよう例外を握りつぶす。
- デバッグが必要な場合に備え、console出力は最小限にする。

### テスト方針

- `manifest.json` が正しいJSONとして読めることを確認する。
- `content.js` が構文エラーなしで読めることを確認する。
- Chromeの拡張機能管理画面で `app/` を読み込めることを確認する。
- DAZNの動画再生ページで、プレイヤーが横幅いっぱいに近づくことを手動確認する。

- ポップアップ / オプション画面。
- Chrome Web Store公開。
- DAZN固有DOMを前提にした精密なセレクタ調整。

### 後で検討すること

- DAZN実画面のDOMに合わせたセレクタ最適化。
- ON/OFF切り替え。
- プレイヤー高さやサイドバー非表示などの追加UI改善。
- Vite / TypeScriptへの移行。

## AIエージェント向け実装ルール

- 初期MVPでは依存パッケージを追加しない。
- DAZNの認証、通信、DRM、動画データには介入しない。
- content scriptはページ表示調整に限定する。
- DAZN固有DOMが不明な箇所は、汎用実装として最小限に留める。
- 実機確認できない項目は、検証ドキュメントに未確認として明記する。

## 追加対応方針

### 2026-06-15: ログイン後再生ページのサイドメニュー干渉修正

- `sample/dazn-videopage-logon-20260615/` のログイン後DOMを参照し、DAZN側の `[data-target="player-layout"]` と右側パネル / プレイヤー内サイドバーの表示領域を尊重する。
- プレイヤー幅は `window.innerWidth` や `100vw` 固定ではなく、実際のプレイヤーレイアウト幅を基準に計算する。
- サイドメニュー展開中は、動画最大化よりも見切れ・重なり防止を優先し、安全に表示できる幅へ収める。
- DAZNのCSS moduleクラス名は変わりやすいため、`data-target` / `data-test-id` など比較的安定した属性を優先する。

### 2026-06-15: ヘッダー追従固定の解除

- DAZNのヘッダーがスクロールや画面上部へ追従しないよう、拡張適用中は `[data-test-id="HEADER"]` と `header` の `position` を `static` に戻す。
- ヘッダーが `static` になった場合は動画表示領域からヘッダー高さを差し引かず、固定ヘッダーが表示領域を覆う場合だけヘッダー高さを差し引く。

### 2026-06-18: ヘッダーは上部ホバー時のみ表示する

- DAZNのヘッダーは通常時に画面外へ隠し、画面上部へマウスを移動したときだけ短時間表示する。
- ヘッダーはプレイヤーのレイアウト高さとして扱わず、動画プレイヤーは画面最上部から使えるようにする。
- DAZN側がヘッダー分として `margin-top` / `padding-top` を残す場合があるため、拡張適用中はページ本体の大きな上部余白を打ち消しつつ、視認性のためプレイヤー上部に 12px の余白だけ残す。
- DAZN側の動画セクション左右paddingを打ち消し、プレイヤーを横幅いっぱいに近づける。
- ホバー表示中のヘッダーは動画の上に重ねるため、動画サイズ計算では常にヘッダー高さを差し引かない。

### 2026-06-18: 右側固定バーとホバー文字列は非表示にする

- DAZN再生ページ右端の固定バー、開閉ボタン、ホバー時に出る文字列は不要なUIとして扱い、拡張適用中は表示しない。
- `[data-test-id="SIDE_BAR"]` と、DAZNの右端開閉ボタン / tooltip 系クラスをCSSで `display: none` にし、幅・flex-basis・pointer-eventsも無効化する。
- 右側バーをプレイヤー幅計算用に保持しないため、サイドバー幅固定用のCSS変数更新は削除する。
