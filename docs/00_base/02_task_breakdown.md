# Task Breakdown Template

実装方針を、AIエージェントが安全に実行できる作業単位へ分解するためのテンプレートです。  
`docs/00_base/01_implementation.md` を埋めたあと、実装に入る前にこのファイルを作成し、作業順序、変更対象、完了条件、依存関係を明確にします。

## 使い方

1. `docs/00_base/00_idea.md` と `docs/00_base/01_implementation.md` の前提を確認する。
2. 実装スコープを、1つずつ完了判定できるタスクに分ける。
3. 各タスクに、目的、変更予定ファイル、完了条件、確認方法を記録する。
4. タスク間の依存関係を明記し、実装順序を決める。
5. 不明点は `docs/01_pending` に転記する。
6. 合意した分割方針や実装順序は、必要に応じて `docs/02_decision` に転記する。
7. 実装中に見つかった改善指摘は `docs/03_review` に転記する。
8. 実装後の確認項目は `docs/00_base/03_verification.md` に転記する。
