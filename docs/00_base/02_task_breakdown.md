## 実装ゴール

### 今回達成すること

- Chromeにローカル拡張として読み込める `app/` を作る。
- DAZNページ上でcontent scriptとCSSを読み込む。
- 動画プレイヤー周辺の横幅制約を緩め、ウィンドウ横幅いっぱいに近い表示へ寄せる。
- DAZN DOMが未確定でも、`video` 要素起点の汎用実装で検証可能にする。

### 完了後に満たすべき条件

- `app/manifest.json` がManifest V3として成立している。
- `app/content.js` が `video` 要素を検出し、対象要素にCSSクラスを付与できる。
- `app/content.css` が対象要素の横幅制約を上書きできる。
- Chrome拡張機能管理画面から `app/` を読み込む手順が分かる。

### 今回扱わないこと

- ビルド環境導入。
- TypeScript化。
- Chrome Web Store公開。
- ポップアップ、オプション、トグルUI。
- DAZNログイン後DOMを前提にした精密な調整。

## 2: 最小Chrome拡張追加

### 目的

Chromeに読み込める最小構成のManifest V3拡張を追加し、DAZNページでプレイヤー横幅調整を自動適用する。

### 作業内容

- `manifest.json` にDAZNドメイン向けcontent scriptを定義する。
- `content.js` で `video` 要素を探し、対象要素にクラスを付与する。
- `MutationObserver` と `resize` で再適用する。
- `content.css` で対象要素の横幅制約を上書きする。

### 変更予定ファイル

| ファイル | 変更内容 | 新規 / 既存 |
| --- | --- | --- |
| `app/manifest.json` | 拡張定義 | 新規 |
| `app/content.js` | DOM検出とクラス付与 | 新規 |
| `app/content.css` | プレイヤー横幅調整CSS | 新規 |

### 完了条件

- Chromeに `app/` を読み込むためのファイル一式が揃っている。
- content scriptがDAZN URLにのみ適用される。
- `video` 要素が見つかったときに調整用クラスが付与される。

### 確認方法

- `manifest.json` をJSONとして検証する。
- `content.js` の構文チェックを行う。
- 可能であればChromeで `app/` を読み込む。

### 注意点・リスク

- DAZN実DOM次第では、初期CSSだけでは十分に効かない可能性がある。
- 効きすぎて周辺UIが崩れる場合は、DOM情報をもとにセレクタを絞る。

## 4: サイドメニュー干渉修正

### 目的

ログイン後DAZN再生ページで、サイドメニュー / 右側パネル展開時に動画の左端が見切れたり、メニューが動画にかぶったりしないようにする。

### 作業内容

- `[data-target="player-layout"]` の `100vw` 固定と負のmarginを外す。
- プレイヤー幅を `window.innerWidth` ではなく、DAZNの実レイアウト幅を基準に計算する。
- `[data-test-id="SIDE_BAR"]` の表示状態を確認し、動画領域と重なる場合は安全幅に反映する。
- サイドメニュー開閉に追従できるよう、`MutationObserver` で属性変更も監視する。

### 変更予定ファイル

| ファイル | 変更内容 | 新規 / 既存 |
| --- | --- | --- |
| `app/content.js` | 実レイアウト幅基準のサイズ計算、サイドバー検出、属性変更監視 | 既存 |
| `app/content.css` | `100vw` 固定と `overflow-x` 強制を削除 | 既存 |
| `docs/00_base/01_implementation.md` | 追加対応方針を追記 | 既存 |
| `docs/00_base/03_verification.md` | サイドメニュー確認シナリオを追記 | 既存 |
| `docs/02_decision/implementation_policy_decision.md` | 実レイアウト幅基準の決定を追記 | 既存 |

### 完了条件

- サイドメニュー / 右側パネル展開時に、拡張機能がDAZN側のレイアウト幅を尊重する。
- 動画の左端見切れと、メニューの動画への重なりが解消または軽減される。
- `content.js` の構文チェックに成功する。

### 確認方法

- `node --check app/content.js` を実行する。
- Chromeで `app/` を読み込み、DAZNログイン後再生ページでサイドメニュー展開時の表示を目視確認する。

### 注意点・リスク

- サイドメニュー展開中は動画最大化効果が弱くなる可能性がある。
- DAZNのDOM属性が変わった場合は、検出セレクタの再調整が必要になる。
