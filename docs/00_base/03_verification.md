# Verification Template

実装後に、作業が目的・MVP・実装方針を満たしているか確認するためのテンプレートです。  
`docs/00_base/02_task_breakdown.md` の各タスク完了後、または一連の実装完了後にこのファイルを埋め、テスト結果、手動確認、未確認事項、残課題を記録します。

## 使い方

1. `docs/00_base/00_idea.md` の成功条件と MVP を確認する。
2. `docs/00_base/01_implementation.md` のテスト方針と実装スコープを確認する。
3. `docs/00_base/02_task_breakdown.md` のタスクごとの完了条件を確認する。
4. 自動テスト、静的チェック、手動確認を分けて記録する。
5. 確認できなかった項目は、理由と影響を明記する。
6. 未解決事項は `docs/01_pending` に転記する。
7. 合意した検証基準やリリース判断は、必要に応じて `docs/02_decision` に転記する。
8. 不具合・改善指摘・追加対応は `docs/03_review` に転記する。
